# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
make lint      # Run shellcheck and shfmt on all shell scripts
make test      # Run Bats test suite
make check     # Run lint + tests (also what make check runs in CI)
make tools     # Verify required tools are installed (shellcheck, shfmt, bats)
```

To run a single test by name:
```bash
bats test/buildpack.bats --filter "test name here"
```

Required tools: `shellcheck`, `shfmt`, `bats` (install via apt-get or brew).

## Architecture

This is a Heroku buildpack that fetches AWS CodeArtifact authentication tokens during the build and exposes them as environment variables for `uv` (the Python package manager).

**Buildpack scripts (`bin/`):**
- `detect` — Checks `pyproject.toml` exists and contains `[[tool.uv.index]]`; exits 0 (detected) or 1 (skip)
- `compile` — Main logic: loads env vars, logs AWS context (`log_aws_context`), calls `aws codeartifact get-authorization-token`, writes an `export` shell script to the buildpack root with `UV_INDEX_<NAME>_USERNAME=aws` and `UV_INDEX_<NAME>_PASSWORD=<token>`
- `release` — No-op release hook (outputs empty YAML)

**How credentials flow downstream:**
The `export` file generated by `compile` is executed by Heroku before running subsequent buildpacks. This makes the UV credentials available to `heroku/python` which runs `uv sync`/`uv pip install`.

**Intended buildpack order:** `heroku-community/subdir` → `heroku-community/awscli` → this buildpack → `heroku/python`

**Required Heroku config vars:**
- `AWS_CODEARTIFACT_DOMAIN`
- `AWS_CODEARTIFACT_DOMAIN_OWNER`
- `AWS_CODEARTIFACT_REGION`
- `UV_CODEARTIFACT_INDEX_NAME` (optional, default: `"codeartifact"`)

**AWS credential config vars (set one option):**
- Option A (recommended): `AWS_CODEARTIFACT_ACCESS_KEY_ID` + `AWS_CODEARTIFACT_SECRET_ACCESS_KEY` — mapped to standard names before calling the AWS CLI
- Option B: `AWS_ACCESS_KEY_ID` + `AWS_SECRET_ACCESS_KEY` (+ `AWS_SESSION_TOKEN` for temporary credentials)

**Index name normalization:** The index name is uppercased with hyphens replaced by underscores for env var naming (e.g., `private-prod` → `UV_INDEX_PRIVATE_PROD_USERNAME`).

**Tests** use a stub `aws` binary (prepended to `PATH`) that returns `stub-token-123` — no real AWS credentials needed to run the suite. Some tests use a custom failing stub to verify error output.
